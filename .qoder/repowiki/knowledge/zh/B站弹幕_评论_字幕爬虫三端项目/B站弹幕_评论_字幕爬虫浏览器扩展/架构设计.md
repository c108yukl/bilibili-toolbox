扩展采用 MV3 Service Worker + popup/options 三页结构：
- manifest.json 声明 permissions（storage/downloads/activeTab/cookies/contextMenus/notifications）及 host_permissions（*.bilibili.com），入口为 background.js（service_worker）、popup.html/popup.js（用户面板）、options.html/options.js（设置页）。
- background.js 是核心编排器：通过 chrome.runtime.connect(name='scraper') 与 popup 建立长连接，接收 {action:'start'|'cancel'} 并驱动 startTask → handleDanmaku/handleSubtitle/handleComments 流水线；同时监听 contextMenus.onClicked 以无头模式直接下载文件（此时 currentPort=null，send('file') 走 chrome.downloads）。
- utils.js 通过 importScripts 被 background 共享，集中实现 WBI 签名（getWbiKeys + getMixKey + encryptWbi，含 MD5 内联实现与 MIXIN_TABLE 混表）、BV 号提取、时间格式化、弹幕 XML 正则解析以及所有输出生成器（genJSON/genCSV/genTXT/genSRT/genASS/genLRC）。
- popup.js 负责 UI 状态机（running/port/logBox/downloadArea）、自动从当前 tab URL 提取 BV 号、可选读取 .bilibili.com cookies 填充 cookie 字段，并通过 port.postMessage 回传 progress/info/success/error/file/done/abort 事件。
- options.js 维护 DEFAULTS 配置对象，读写 chrome.storage.sync.settings，提供 Cookie 预览刷新。
依赖方向单向：popup/options → background（消息通道），background → utils（函数调用），无任何外部库引入。
