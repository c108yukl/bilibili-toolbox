模块按功能拆分为独立文件并通过 `bilibili/__init__.py` 统一暴露顶层 API：
- 认证层 `auth.py`：将用户 Cookie 字符串解析为 `bilibili_api.Credential`；
- 工具层 `utils.py`：纯函数（如 `extract_bvid`）无外部依赖；
- 业务层 `danmaku.py` / `comments.py` / `subtitle.py`：封装 `bilibili_api.video`、`comment`、`ass` 等客户端，负责分页、楼中楼、语言选择等调用逻辑；
- 存储层 `cache.py`：基于 `.bili_cache/` 目录的 JSON 文件缓存，以 MD5(bvid:dtype:page) 为键，带 `_cached_at` 时间戳过期；
- 输出层 `formatters.py`：将原始数据转为 txt/json/csv/srt/ass/lrc 并写入项目根目录。
依赖方向单向：业务层 → cache/formatters + bilibili_api，`__init__.py` 仅做再导出，不引入额外耦合。所有 I/O 操作均为 async，通过 `asyncio.sleep` 控制请求频率。
