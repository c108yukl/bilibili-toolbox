本模块由三个并列的入口脚本组成，均复用同一套异步抓取函数（来自 `bilibili` / `bilibili_demo` 包），按交互形态分层：
- `cli.py`：基于 `argparse` 的纯命令行入口，负责参数解析、默认行为推导（未指定操作时全部执行）、调用 `bilibili` 包的 `get_danmaku/get_comments/get_all_comments/get_subtitle` 等异步 API，并通过 `bilibili.cache.CACHE_DIR` 输出缓存路径。
- `app.py`：Streamlit 本地网页版，侧边栏收集 BV 号、功能开关、页数、语言、保存格式、Cookie、缓存开关等参数，内部用自定义 `CaptureIO` + `redirect_stdout` 把底层 print 重定向到 `st.code` 日志区，并用 `asyncio.run(run())` 在 Streamlit 请求生命周期内运行协程；完成后扫描同级目录生成 `st.download_button` 供下载。
- `bilibili_demo.py`：自包含的单文件可执行 demo，内置完整的弹幕/评论/字幕抓取逻辑、`.bili_cache` JSON 缓存、`save_danmaku/save_comments/save_subtitle` 多格式落盘、`parse_cookie` 解析 SESSDATA 并构造 `bilibili_api.Credential`，同时自带一套与 `cli.py` 平行的 `argparse` 子集（含 `--danmaku-only/--comments-only` 扩展）。
依赖方向：两个 UI 层 → `bilibili` 或 `bilibili_demo` 提供的异步 API；`bilibili_demo.py` 直接依赖 `bilibili_api`、`aiohttp`，不反向依赖 UI 层。
